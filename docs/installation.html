安装指南

前提条件

在开始安装前，请确保您的系统满足以下条件：

  • Python 3.7+
  • pip (Python包管理器)
  • 如果使用Docker方式部署，则需要Docker和Docker Compose

方式一：使用Docker（推荐）

这是最简单的部署方式，直接使用我们预构建的官方镜像：

1

确保安装了Docker和Docker Compose

如果您还没有安装Docker，请访问Docker官方网站获取安装指南。

2

创建项目目录

mkdir -p FeedForge/{rss_files,rss_output,logs,docker_data}
cd FeedForge
3

下载docker-compose配置文件

curl -O https://raw.githubusercontent.com/binbin1213/FeedForge/main/docker-compose.hub.yml
4

启动容器

docker-compose -f docker-compose.hub.yml up -d
5

访问应用

在浏览器中访问 http://localhost:8080 开始使用

6

停止应用（需要时）

docker-compose -f docker-compose.hub.yml down

方式二：从源码安装

如果您想自定义安装过程或修改代码，可以选择从源代码安装：

1

克隆仓库到本地

git clone https://github.com/binbin1213/FeedForge.git
cd FeedForge
2

创建并激活虚拟环境（推荐）

python -m venv venv
# Windows
venv\Scripts\activate
# macOS/Linux
source venv/bin/activate
3

安装依赖

pip install -r requirements.txt
4

安装可选依赖（用于通知系统）

# 桌面通知支持（跨平台）
pip install plyer==2.1.0

# Windows桌面通知支持（仅Windows系统需要）
pip install win10toast==0.9
5

启动应用

python app.py
6

访问应用

在浏览器中访问 http://localhost:8080 开始使用

方式三：从源代码构建Docker镜像

如果您想自定义Docker镜像或修改代码，可以选择从源代码构建：

1

克隆仓库

git clone https://github.com/binbin1213/FeedForge.git
cd FeedForge
2

使用Docker Compose构建并启动

docker-compose up -d --build
3

访问应用

在浏览器中访问 http://localhost:8080 开始使用

4

停止应用（需要时）

docker-compose down

Docker数据持久化

FeedForge (飞阅) 容器将以下目录挂载到宿主机，确保数据持久化：

  • ./rss_files：存储生成的RSS源文件
  • ./rss_output：存储输出文件
  • ./logs：应用日志
  • ./docker_data/rss_feeds.db：SQLite数据库文件

您可以根据需要备份这些目录。

Docker环境变量

Docker部署支持以下环境变量配置：

环境变量 说明 默认值
TZ 容器时区 Asia/Shanghai
PORT 应用端口 8080

可以在docker-compose.yml文件中的environment部分修改这些设置。
